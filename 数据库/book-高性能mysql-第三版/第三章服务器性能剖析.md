# 服务器性能剖析
性能相关的灵魂三问
* 如何确认服务器是否达到了性能最佳的状态
* 找出某条语句为什么执行不够快
* 诊断被用户描述成"停顿"、"堆积"或者"卡死"的某些间歇性疑难故障

有一个简单方法：专注于测量服务器时间花费在哪里
技术：性能剖析(profiling)

# 3.1性能优化简介

本书关于性能定义：完成某件任务所需要的时间度量(也即响应时间，注意和吞吐量的区别)

本书数据库服务器目的是执行SQL语句，它关注的任务是查询/语句.
性能用查询(包含DDL,DML)响应时间来度量，单位是每个查询花费的时间。
